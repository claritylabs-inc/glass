# Glass Primitives

Use this skill before adding a shared abstraction or when a task risks duplicating an existing owner. The canonical catalog is in `AGENTS.md` under `Primitive Catalog And Reuse Rules`; keep this skill and that section aligned.

## Rule

Search before creating:

```bash
rg "<concept|route|component|table|helper>" AGENTS.md components convex lib hooks app extraction-worker imessage-worker
```

Prefer extending the existing primitive when the meaning matches. Add a new primitive only when reusing the old one would make the API misleading. If a change creates, deletes, renames, or materially changes a primitive, update both `AGENTS.md` and this skill in the same PR.

Select and dropdown popup content stays on the shared portaled overlay path so overflow containers cannot clip it. Nested editors aggregate into one page-level save status. Human-facing absolute dates use `MMM D, YYYY` through the shared date-format owner; native date inputs, storage/API values, relative ages, and time-only diagnostics keep their domain-specific formats.
- Branding: rendered app/org marks live in `components/ui/brand-icon.tsx`, `components/ui/org-brand-icon.tsx`, and `components/ui/logo-icon.tsx`; browser theme tokens and color utilities live in `lib/branding.ts`; Next.js server-only viewer metadata lives in `lib/viewer-branding.ts`; Convex-safe branding context and white-label gates live in `convex/lib/branding.ts`; Convex org-logo URL attachment lives in `convex/lib/orgBranding.ts`; shared email shells live in `convex/lib/emailTemplate.ts`; notification-email composition lives in `convex/lib/notificationEmailTemplate.ts`.

- Local-first sync and current org: `lib/sync/use-cached-query.ts`, `lib/sync/glass-cached-queries.ts`, `lib/sync/operator-cached-queries.ts`, `lib/sync/use-local-first-auto-save.ts`, `lib/sync/auto-save-sequencer.ts`, `hooks/use-current-org.tsx` for the canonical lightweight viewer-org `useCurrentOrg`, and `lib/hooks/use-active-org-context.ts` for URL-aware selected-org surfaces. `useLocalFirstAutoSave` is the sole save-on-change primitive; validation-aware saves use `enabled`/`canSave` and custom timing uses `delayMs`. Settings text and address inputs keep `autoSave` disabled and call `saveNow` after blur; `AddressAutofillInput` forwards `onBlur` and reports completed Mapbox values through `onRetrieve`. Availability-checked inputs pause auto-save while focused and resume after blur, while discrete selects flush after selection. `applyLocal` synchronizes shared cache only after the current backend intent succeeds, and dependent actions await `saveNow`. The shared sequencer keeps backend flushes in intent order, while the hook suppresses stale value/reset callbacks; `canSave` gates permission and validation instead of dirty state, and identity or availability fields use current raw input rather than debounced validation input for `valueKey`. Auto-save writes bypass the durable outbox so stale form intents cannot replay after reload; failed writes require explicit retry. Nested editors report into one page-level `AutoSaveStatus`; completion stays inline and the hook owns the deduplicated error toast, so do not add duplicate Saved indicators or success toasts. Reserve `saveNow({ force: true })` for derived legacy state that began as the local baseline but still needs persistence. Do not add page-local debounced save effects.
- Auth/access: `convex/lib/access.ts`, `convex/lib/operatorIdentity.ts`, `convex/lib/apiAuth.ts`, `convex/oauth.ts`, Convex HTTP OAuth handlers, and `convex/lib/threadAccess.ts`.
- Model routing: `extraction-worker/src/modelRoutingPolicy.ts`, `convex/lib/modelCatalog.ts`, `convex/lib/models.ts`, and model/provider UI logo primitives. Operator backfill calls `runCoverageRecovery` over stored spans/nodes without reparsing. Both recovery paths remove premium/exposure values from coverages, separate premium, taxes, fees, and total payable, and store referenced auto/property/location schedules in `coverageSchedules` with source provenance; scheduled assets do not belong in description of operations. Every recognized extracted date destination, including nested profiles, coverages, forms, endorsements, declarations, and supplementary facts, passes through `normalizeExtractedDateFields` at the mapping and persistence boundaries and stores parseable dates as `MM/DD/YYYY`. Cards retain content-width breakpoints of one column below 48rem, two columns from 48rem, and three from 64rem; coverage schedules live in the Coverages tab. `policyPartyContext` owns manual-override, structured-policy, compatibility, and legacy precedence and limits client-profile fallback to insured address and operations. `coverageBreakdown` preserves coverage provenance while owning grouped LOB schedules, referenced asset schedules, and flat-order compatibility.
